Virtual Memory Allocation using mmap()

What is Virtual memory?

Virtual memory is a memory management technique that provides an "idealized abstraction of the storage resources that are actually available on a given machine," which "creates the illusion to users of a very large (main) memory."

Pages

Virtual memory is divided into fixed-size blocks called pages. Similarly, physical memory (RAM) is also divided into the same-sized blocks.

Page Frame:

The corresponding unit in physical memory is called a page frame.

Page Table

A page table is used to keep track of the mapping between the virtual addresses in the program and the actual physical addresses in RAM.

mapping

Each entry in the page table represents a mapping between a virtual page and its corresponding physical page frame.

Page falts

When a program accesses a part of virtual memory that is not currently in physical memory (page fault), the operating system intervenes.

Handling Page Faults

The operating system brings the required page into physical memory from the disk. If there's not enough space in RAM, it may need to swap out another page to the disk to make room.

Swapping

Swapping involves moving an entire process or a part of it between RAM and disk. When the operating system needs to free up space in RAM, it may swap out some pages to the disk. Performance Impact: Swapping can introduce overhead, as moving data between RAM and disk is slower than accessing data in RAM directly.

Benefits of Virtual Memory Page Allocation

Effective Use of Resources: Virtual memory allows programs to use more memory than physically available, as the system can temporarily store less frequently used data on disk. Isolation: Each process has its own virtual address space, providing isolation from other processes. In summary, virtual memory page allocation is a mechanism that allows a computer to efficiently manage its memory resources by using a combination of RAM and disk space. It enables the illusion of a large, contiguous block of memory for programs while efficiently utilizing physical memory through techniques like demand paging and swapping.
